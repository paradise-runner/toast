// capabilities

▦ multi-tab buffers
dirty-state tracking, close buttons, and quit confirmation
▴ tree-sitter parsing
syntax highlighting for Go, Python, JS/TS, Rust, CSS, HTML, YAML, Bash, and Markdown
◈ LSP integration
completions, hover docs, diagnostics, and definition lookups for configured languages
⌕ ripgrep search
project-wide search via rg plus in-file find and replace
⊕ git-aware tree
status indicators, ignored-file dimming, create/delete actions, and draggable resizing
⚙ editor workflow
go to line, markdown preview, external file watching, and binary file protection
◉ theme system
built-in system, toast-dark, and toast-light themes plus VSCode theme importing
▷ rope-backed buffer
full undo and redo with editor state managed entirely in terminal
◯ config file
~/.config/toast/config.json for theme, editor, sidebar, and LSP settings
⇅ file-first startup
open directories, existing files, or a new file path with git-root detection

// install and run

launch
  • toast opens the current directory
  • toast path/to/dir opens a specific project
  • toast path/to/file opens a file and detects the git root
  • toast new/file.go opens a new buffer if the parent directory exists
requirements
  • rg is required for project search
  • gopls, pyright-langserver, typescript-language-server, and rust-analyzer enable LSP features when installed
  • toast migrate-theme vscode path/to/theme.json imports a VSCode theme
